# Day 13 知识点总结：RAG Web 界面

> 阶段：阶段四 - RAG 检索增强生成实战

---

## 核心概念

- **文件上传**：前端上传文档，后端处理（分块→Embedding→存储）
- **检索结果可视化**：展示"参考文档"来源
- **整合到 Chat**：在原有聊天界面增加 RAG 能力

## 功能实现

### 后端流程

```
上传文件 → 保存到服务器 → 提取文本 → 分块 → Embedding → 存储到向量库
```

### 前端流程

```
用户提问 → Chat API → RAG 检索 → 拼接上下文 → LLM 生成 → 展示回答 + 引用来源
```

## UI 设计

```
Chat 界面（原有）
└── AI 消息
    ├── 回答内容
    └── 引用来源（可折叠）
        ├── 📄 doc1.md - chunk #3
        └── 📄 doc2.md - chunk #1
```

## 实践要点

- 文件上传用 Ant Design 的 `Upload.Dragger`
- 后端需要处理多种格式（MD、TXT、PDF）
- 引用来源用折叠面板展示，避免干扰阅读

## 一句话总结

> RAG Web 界面 = 上传文档建立知识库 + 聊天时自动检索相关上下文。
