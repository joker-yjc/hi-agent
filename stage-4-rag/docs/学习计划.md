# 前端开发者 LLM & Agent 开发学习计划

> 目标：以最低学习成本，从前端开发者过渡到 AI 应用开发者
> 技术路线：以 TypeScript/JavaScript 生态为主，Python 为辅
> 更新时间：2026-03
> 学习模式：每个阶段按天拆分 → 每阶段末提交 Demo → 由 Qoder 验收

---

## 整体路线概览

```
阶段一（Day 1-2）  ：LLM 基础认知 & Prompt Engineering
       ↓
阶段二（Day 3-5）  ：LLM API 集成开发（TypeScript）
       ↓
阶段三（Day 6-9）  ：构建 AI Chat Web 应用
       ↓

       ↓

       ↓

       ↓
```

> Day 的定义：一个学习单元，不一定是自然日。你可以根据自己的时间灵活安排，
> 半天完成一个 Day 或两天完成一个 Day 都没问题，核心是**完成该 Day 的任务**。

#### 向量存储方案选择（三选一）

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **内存存储（学习用）** | 零依赖，最快上手 | 重启丢失，不适合生产 | 入门首选 |
| **Upstash Vector** | 免费额度，Serverless | 需要注册账号 | 进阶推荐 |
| **PostgreSQL + pgvector** | 生产级，功能完整 | 需要安装 PG | 生产推荐 |

## 注意事项

1. **优先用 TypeScript 生态**：不要一开始就学 Python 的 LangChain，JS/TS 生态已经足够成熟
2. **先跑通再优化**：先用 API 跑通流程，再考虑本地模型、性能优化等
3. **API 费用控制**：学习阶段用 `gpt-4o-mini` 或 DeepSeek 控制成本，或用 Ollama 本地模型
4. **保持关注更新**：AI 领域变化极快，关注 Vercel AI SDK、MCP 协议的最新版本
5. **动手为主**：每个概念都要写代码验证，不要只看文档
6. **每个阶段做完就找我验收**：完成一个阶段的 Demo 后，直接告诉我"阶段 X 完成了"，我会帮你检查
