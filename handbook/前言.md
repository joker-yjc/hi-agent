# 前言

> 为什么我要花时间整理这份手册？
> 以及你应该怎么使用它。

---

## 问题的起源

作为一个前端开发工程师，我决定系统学习 LLM 和 Agent 开发。我发现最大的困难不是"学不会"，而是**找不到可靠的信息来源**。

三个典型场景：

1. **想查一下 `generateObject` 的用法** → 搜索到一堆 LangChain Python 的代码，完全对不上我的 TypeScript 技术栈
2. **想理解 Agent 记忆系统** → 看了 5 篇文章，每篇对"短期记忆"和"长期记忆"的定义都不一样，越看越懵
3. **想做一个 RAG 问答系统** → 教程说的 PostgreSQL + pgvector，我只想先跑通一个 demo 看看效果

结论是：**前端生态的 AI 开发中文资料存在严重缺口**。要么得依靠大量的英文原版资料，要么就得自己在信息的汪洋中检索、测试与验证。

## 这份手册的定位

它不是：
- ❌ 一份面面俱到的教材（hello-agents 已经做得很好，去读那个）
- ❌ 一份 API 文档的翻译版（Vercel AI SDK 官方文档很全，但本质仍是英文的 interface）

它是：
- ✅ 一份**经过代码验证**的知识速查手册
- ✅ 一份**前端开发者视角**的技术选型参考
- ✅ 一份**踩坑实录**，帮你避开那些网上教程不会告诉你的问题

## 你应该怎么用

**如果你在学某个阶段**（比如正在做 Day 14-16 的 Agent 任务）：
- 打开对应章节，先看"概念速览"建立认知
- 看"代码骨架"知道关键调用怎么写
- 参考"实战建议"规划实现步骤
- 最后看"踩坑记录"避免重复犯错

**如果你只是查一个概念**（比如突然忘了 `maxSteps` 的作用）：
- 直接跳到对应章节看"概念速览"
- 用"延伸阅读"里你之前标记过的链接

**如果你在规划自己的 AI 项目**：
- 看"技术选型"板块，了解前端生态下各方案的优劣
- 看"代码骨架"快速搭建原型
- 需要完整实现时，看同仓库 `stage-*/` 的成熟代码

## 知识验证原则

一个知识点能被写进这本手册，至少通过以下一项验证：

1. ✅ **自己写代码跑通过** — 有对应的 `stage-*/` 脚本可直接运行复现
2. ✅ **对比过官方源码** — 看过 `@ai-sdk/*` 的 GitHub 源码确认行为
3. ✅ **在某个特定模型上反复验证** — 比如通义千问的 JSON 输出问题
4. 📖 **来自权威官方文档** — 标记为引用，非个人经验

没有亲自验证过的概念，**不写进"概念速览"**，最多在"延伸阅读"留个链接。

## 技术栈约定

本手册基于以下技术栈（截至 2026 年 4 月）：

| 技术 | 版本 | 备注 |
|------|------|------|
| Vercel AI SDK | 6.x | `generateObject` 已废弃，改用 `generateText` + `Output.object` |
| Next.js | 16 / 14 | Stage 3 用 16，Stage 4 App 用 14 |
| React | 19 / 18 | 跟随 Next.js 版本 |
| TypeScript | ^5 | strict 模式 |
| pnpm | workspace | monorepo 统一管理 |
