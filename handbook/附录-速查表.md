## API 调用速查

| 需求 | 代码 |
|------|------|
| 基础文本生成 | `generateText({ model, prompt })` |
| 流式输出 | `const r = streamText({ model, prompt }); for await (const c of r.textStream)` |
| 多轮对话 | `const msgs = [{role:'system',content:'...'}, ...]; generateText({ model, messages: msgs })` |
| 结构化输出 | `generateText({ model, output: Output.object({ schema: zSchema }), prompt })` |
| 批量 Embedding | `embedMany({ model, values: ['a','b','c'] })` |
| 单次 Embedding | `embed({ model, value: 'text' })` |
| 相似度 | `cosineSimilarity(vecA, vecB)` (0-1, 越接近 1 越相似) |

## 记忆模式速查

| 模式 | 何时用 | 一句话实现 |
|------|--------|-----------|
| Buffer | 短对话（<10 轮） | `messages.push(...)` 不处理 |

## 常见 Provider 陷阱速查

| 问题 | 解决 |
|------|------|
| `@ai-sdk/alibaba` 不支持 Embedding | 改用 `@ai-sdk/openai` 兼容模式 |
| 阿里云结构化输出有时不严格 | `prompt` 里加 "JSON" 关键词 |
| `generateObject` 报 deprecated | 改用 `generateText` + `Output.object()` |
| 阿里云 API 401 | 检查 baseURL 是否设为国内地址 |
| `useChat` 闭包里 modelId 不更新 | 用 `useRef` 存 modelId，不用 `useState` |
| `transport.body` 闭包用 stale state | 同上，关键变量用 `useRef` |
| 切换会话后 messages 不更新 | 调 `setMessages(session?.messages ?? [])` |
| SSR 报 `localStorage is not defined` | 加 `typeof window === 'undefined'` 守卫 |

## 模型路由速查（第 18 章）

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 简单问答 / 闲聊 | qwen-turbo / gpt-4o-mini | 便宜，响应快 |
| 代码分析 / 复杂推理 | qwen-max / gpt-4o | 质量高 |
| 路由分类 / 意图判断 | qwen-turbo | 只做分类，不需要大模型 |
| Embedding | text-embedding-v3 | 向量质量和成本的平衡 |
| Vision（图片理解） | qwen-vl-max / gpt-4o | 需要视觉能力 |
| 图片生成 | dall-e-3 | 目前 AI SDK 支持最好 |

## AI 安全防护速查（第 21 章）

```
防护四件套（从外到内）：
1. 输入过滤 → 正则匹配注入模式（detectInjection）
2. System Prompt 防护 → 加自保指令（SAFETY_PREFIX）
3. 输出过滤 → 检测敏感信息泄露（filterOutput）
4. 速率限制 → 按 IP 限制请求频率（RateLimiter）

关键原则：
- 不要在错误信息中暴露检测逻辑
- 没有银弹，多层策略组合使用
- 流式输出下的过滤只能事后检查
```

## 多 Agent 模式速查（第 22 章）

| 模式 | 代码结构 | 何时用 |
|------|---------|--------|
| Supervisor | Router → 结构化输出选 Agent → 调用对应 Agent | 多意图分流 |
| Pipeline | Agent A 输出 → Agent B 输入 → Agent C 输入 | 有明确阶段的任务 |
| Parallel | Promise.all([AgentA, AgentB, AgentC]) → 汇总 Agent | 多角度分析 |
| 单 Agent | 一个好的 System Prompt + 几个 Tool | 大多数场景够用 |

## LLM Wiki 流程速查（第 24-25 章）

```
构建流程：
  raw/ (原始素材) → llmwiki compile → wiki/ (结构化知识)
                                         ↓
                                   embedMany → 向量索引
                                         ↓
                                   embed(query) → 搜索 → LLM 回答

增量更新：

  2. 只对变化文件重新编译
  3. 只对变化的 Wiki 页面重新 Embedding
```
