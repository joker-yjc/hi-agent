4. 用 AI SDK 生成 Embedding

#### 余弦相似度原理

```
向量 A = [a1, a2, a3, ...]
向量 B = [b1, b2, b3, ...]

余弦相似度 = (A · B) / (|A| × |B|)
            = Σ(ai × bi) / (√Σ(ai²) × √Σ(bi²))

公式理解：
- 分子：向量点积，衡量方向一致性
- 分母：两个向量的模长乘积，归一化到 [-1, 1]

结果解读：
- 接近 1：方向相同，语义相似
- 接近 0：方向正交，无关
- 接近 -1：方向相反，对立
```

#### ReAct 模式原理

```
┌─────────────────────────────────────────────────────────┐
│                      ReAct 循环                        │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  用户输入：查询天气                                       │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  Think（思考）：用户想知道北京的天气，需要调用天气API     │
│  Action（行动）：调用 get_weather 工具，参数 city="北京"  │
│  Observation（观察）：获取到天气数据：晴，25°C           │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  Think（思考）：已获取天气信息，可以回答用户了           │
│  Action（行动）：生成最终回答                             │
│  Observation（观察）：回答已生成                         │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  最终回答：北京今天天气晴朗，气温25°C                    │
└─────────────────────────────────────────────────────────┘

ReAct = Reasoning（推理）+ Acting（行动）
核心：LLM 生成 Thought → Action → Observation 的循环，直到得到答案
```

#### MCP vs Tool vs API 对比

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│    维度      │  Tool Calling │    MCP      │    API      │
├──────────────┼──────────────┼──────────────┼──────────────┤
│  定义位置    │ 代码中硬编码  │ 独立服务     │ HTTP端点     │
│  动态性      │ 静态         │ 运行时发现   │ 静态         │
│  协议标准    │ 无（各厂商自定）│ 统一标准    │ REST/GraphQL│
│  上下文共享  │ 无           │ 有（Host）   │ 无           │
│  适用场景    │ 单应用内工具 │ 跨应用扩展   │ 外部服务     │
│  配置方式    │ 代码         │ JSON配置     │ URL调用      │
└──────────────┴──────────────┴──────────────┴──────────────┘

层级关系：
API > MCP > Tool Calling
- API：最底层，通用 HTTP 调用
- MCP：基于 API 的标准化协议，支持发现和上下文
- Tool Calling：LLM 层面的工具抽象，可能基于 MCP 或直接调用 API
```

  | 维度 | 单 Agent | 多 Agent |
