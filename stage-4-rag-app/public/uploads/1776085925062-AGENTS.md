# AGENTS.md - Flutter Learning Project Guidelines

This is a Flutter learning repository containing educational code examples, tutorials, and documentation for learning mobile development with Flutter and Dart.

## 1. Build / Lint / Test Commands

### Running Dart Code
```bash
# Run a single Dart file
dart run lib/main.dart

# Run with debugging
dart run --observe lib/main.dart
```

### Flutter Commands (when Flutter project is created)
```bash
# Create new Flutter project
flutter create project_name
cd project_name

# Run the app
flutter run

# Run on specific device
flutter run -d <device_id>

# Hot reload (in debug mode, press 'r')
flutter run

# Build for release
flutter build apk --release  # Android
flutter build ios --release   # iOS

# Build for web
flutter build web
```

### Linting
```bash
# Run Flutter analyzer
flutter analyze

# Analyze specific file
flutter analyze lib/main.dart

# Fix automatically
flutter analyze --fix
```

### Testing
```bash
# Run all tests
flutter test

# Run a single test file
flutter test test/widget_test.dart

# Run tests matching a pattern
flutter test --name "myTestName"

# Run tests in a specific directory
flutter test test/unit/

# Run with code coverage
flutter test --coverage

# Watch mode (rerun on file changes)
flutter test --watch
```

### Code Generation
```bash
# Generate code (e.g., for json_serializable, hive, etc.)
flutter pub run build_runner build

# Delete conflicting outputs and regenerate
flutter pub run build_runner build --delete-conflicting-outputs

# Watch for changes
flutter pub run build_runner watch
```

## 2. Code Style Guidelines

### General Principles
- Write clean, readable code suitable for learning purposes
- Add Chinese comments to explain concepts (this is an educational repository)
- Use meaningful variable and function names
- Keep functions focused and small (single responsibility)

### File Organization
```
lib/
├── main.dart                 # App entry point
├── models/                   # Data models
├── providers/               # State management (Provider)
├── screens/                 # Screen/Page widgets
├── widgets/                 # Reusable widgets
├── services/                # API and business logic
└── utils/                   # Helper functions

test/
├── unit/                    # Unit tests
├── widget/                  # Widget tests
└── integration/            # Integration tests
```

### Imports
```dart
// Order imports:
// 1. Dart SDK imports
import 'dart:async';
import 'dart:io';

// 2. Package imports (alphabetical)
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

// 3. Relative imports (grouped)
import '../models/user.dart';
import '../services/api_service.dart';

// 4. Export files (if needed)
export 'constants.dart';
```

### Naming Conventions

| Element | Convention | Example |
|---------|------------|---------|
| Files | snake_case | `user_service.dart`, `home_screen.dart` |
| Classes | PascalCase | `class UserService`, `class HomeScreen` |
| Functions | camelCase | `void fetchUser()`, `String formatDate()` |
| Variables | camelCase | `final userName`, `var isLoading` |
| Constants | camelCase or SCREAMING_SNAKE | `const maxSize`, `const API_BASE_URL` |
| Private members | underscore prefix | `_internalMethod`, `_privateField` |
| Enums | PascalCase | `enum UserStatus { active, inactive }` |

### Types and Type Safety
```dart
// Prefer explicit types for class members
class User {
  final String name;      // Explicit type
  final int age;          // Explicit type
  final List<String> tags; // Explicit type for collections
}

// Use 'var' for local variables when type is obvious
var users = getUsers();          // OK - type is clear
var name = 'John';               // OK - inferred as String

// Use 'final' by default, 'var' only when reassignment is needed
final user = User();            // Preferred
var counter = 0;                // Only when changing

// Use 'const' for compile-time constants
const apiKey = 'abc123';
const defaultTimeout = Duration(seconds: 30);
```

### Null Safety
```dart
// Always use null-safe types
String? nullableName;     // Nullable
String nonNullableName;   // Non-nullable (preferred)

// Use null-aware operators
final name = user.name ?? 'Unknown';
final length = user.name?.length ?? 0;

// Use late for delayed initialization
late final User user;
Future<void> init() async {
  user = await fetchUser();
}
```

### Error Handling
```dart
// Use try-catch for async operations
Future<void> loadData() async {
  try {
    final data = await api.fetchData();
    // Handle success
  } on NetworkException catch (e) {
    // Handle specific error
    print('Network error: $e');
  } on ServerException catch (e) {
    // Handle server error
    print('Server error: $e');
  } catch (e) {
    // Handle any other error
    print('Unexpected error: $e');
  }
}

// Use Result type pattern for operations that can fail
Result<User, String> getUser(int id) {
  if (id <= 0) {
    return Err('Invalid user ID');
  }
  return Ok(User(id: id));
}
```

### Widget Best Practices
```dart
// Use const constructors when possible
const MyWidget({Key? key}) : super(key: key);

// Prefer StatelessWidget for pure presentation
class ProfileCard extends StatelessWidget {
  final String name;
  final String email;

  const ProfileCard({
    super.key,
    required this.name,
    required this.email,
  });

  @override
  Widget build(BuildContext context) {
    return Card(
      child: Text('$name - $email'),
    );
  }
}

// Use StatefulWidget only when state is needed
class Counter extends StatefulWidget {
  const Counter({super.key});

  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;  // Private field with underscore

  void _increment() {  // Private method
    setState(() {
      _count++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: _increment,
      child: Text('Count: $_count'),
    );
  }
}
```

### State Management (Provider Pattern)
```dart
// Model with ChangeNotifier
class UserProvider extends ChangeNotifier {
  User? _user;
  
  User? get user => _user;
  
  Future<void> loadUser() async {
    _user = await fetchUser();
    notifyListeners();  // Always call after state change
  }
}

// Using in widgets
class UserScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    // watch: rebuilds when value changes
    final user = context.watch<UserProvider>().user;
    
    return Text(user?.name ?? 'Loading...');
  }
}

class LoginButton extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    // read: just reads value, no rebuild
    return ElevatedButton(
      onPressed: () => context.read<UserProvider>().login(),
      child: Text('Login'),
    );
  }
}
```
